作成日: 2026-06-29

---

## 1. 我々はなぜここにいるのか（Why Are We Here?）

バイクのマスツーリング愛好者は、仲間との思い出を共有するために、ツーリングのハイライトを30分程度の動画にまとめてYouTubeの非公開動画として共有している。しかし6時間超の走行映像からハイライトを探して編集する作業は手間も時間も非常にかかる。

このプロダクトは、AIがハイライトを自動的に検出し動画を生成することで、編集の負担を大幅に削減する。AI技術が成熟した今がまさに実現のタイミングである。

## 3. パッケージデザイン（Product Box）

**キャッチコピー：** ツーリングVlogをAIで自動作成

**推しポイント：**

1. 仲間との会話から盛り上がりポイントを自動検出して動画に抽出
2. ツーリングの全工程をGPS記録からわかりやすく映像化
3. 360°映像から自動で注目ポイントをトラッキング

**「欲しい！」の瞬間：** ドラレコやInsta360で撮りっぱなしの映像が溜まっているとき、YouTubeでこのプロダクトの自動編集結果を見たら必ず欲しくなる。

## 6. 解決策を描く（Show Your Solution）

**基本方針：** ストレージ＆レンダリングはローカル、AI処理はクラウド（ハイブリッド構成）

**技術スタック：** Python 3.11+ / FFmpeg / MoviePy / CLIパイプライン + YAML設定

**処理パイプライン：**

```
01_ingest.py        # ファイル取り込み・整理
02_extract_gps.py   # ドラレコ(EDR-22G)からGPS・時刻情報を抽出
03_extract_audio.py # アクションカム動画から音声トラックを分離
04_analyze.py       # faster-whisper(ローカル)で文字起こし → Claude Haiku APIでハイライト判定
05_select.py        # ハイライト区間の選定・カメラ選択（前/後/360°）
06_map_render.py    # GPS情報からpy-staticmapsでMAP動画を生成
07_reframe_360.py   # FFmpeg v360フィルターで360°映像をリフレーム
08_compose.py       # MoviePyで最終動画を合成・レンダリング
```

各ステップは独立・冪等で、途中から再実行可能。

**入力：** ドラレコ動画（EDR-22G）、Insta360映像（X3）、インカム音声（B+COM 6XR）
**出力：** 編集済みハイライト動画（約30分）
